# Minecraft Education Coding Curriculum for Kids
## 24 Sessions × 45 Minutes | Aligned with Singapore MOE Standards

---

## Curriculum Design Analysis

### Learning Weight Distribution

This curriculum adapts Microsoft's official "Computing with Minecraft" 10-unit curriculum while maintaining proportional learning weights aligned with Singapore's computational thinking standards.

| Topic | Microsoft Unit | Original Hours | New Sessions | New Time | Weight |
|-------|---------------|----------------|--------------|----------|--------|
| Introduction & Interface | Unit 1 | 3 hrs | 2 sessions | 90 min | 8.3% |
| Events | Unit 2 | 3 hrs | 2 sessions | 90 min | 8.3% |
| Coordinates (X, Y, Z) | Unit 3 | 3 hrs | 2 sessions | 90 min | 8.3% |
| Iteration (Loops) | Unit 7 | 3 hrs | 3 sessions | 135 min | 12.5% |
| Conditionals | Unit 5 | 3 hrs | 3 sessions | 135 min | 12.5% |
| Variables | Unit 4 | 3 hrs | 2 sessions | 90 min | 8.3% |
| Functions | Unit 6 | 3 hrs | 2 sessions | 90 min | 8.3% |
| Arrays & Data | Unit 8 | 3 hrs | 2 sessions | 90 min | 8.3% |
| AI & Digital Citizenship | Unit 9 | 3 hrs | 1 session | 45 min | 4.2% |
| Final Project | Unit 10 | 3 hrs | 5 sessions | 225 min | 20.8% |
| **Total** | | **30 hrs** | **24 sessions** | **1080 min** | **100%** |

**Design Rationale:**
- Complex topics (Loops, Conditionals) receive enhanced weight (+12.5%) for mastery-based learning
- Mathematics integration through 3D coordinate system (unique to Minecraft)
- Functions introduced earlier than Scratch curriculum due to Agent command structure
- Arrays included as Minecraft naturally uses lists (inventory, positions)
- Final project maintains ~20% weight aligned with Singapore's emphasis on application

## Learning Objectives

### Computational Thinking Concepts

| Concept | Definition | Minecraft Application | Mastery Level |
|---------|------------|----------------------|---------------|
| **Sequence** | Ordered steps to complete a task | Agent commands executed in order | Apply independently |
| **Events** | Triggers that start code execution | Chat commands, block interactions | Use multiple event types |
| **Coordinates** | 3D position system (X, Y, Z) | Agent positioning, building locations | Navigate and calculate positions |
| **Loops** | Repeating code multiple times | Building patterns, farming automation | Choose appropriate loop type |
| **Conditionals** | Decision-making based on conditions | Block detection, inventory checks | Implement nested conditions |
| **Variables** | Storing and changing data | Scores, counters, resource tracking | Use strategically |
| **Functions** | Reusable blocks of code | Custom Agent commands | Create parameterised functions |
| **Arrays** | Ordered lists of data | Position lists, item inventories | Manipulate arrays |

### Computational Thinking Practices

| Practice | Definition | Development Across Sessions |
|----------|------------|----------------------------|
| **Experimenting & Iterating** | Developing through trial and error | Sessions 1-4 (introduce), 5-17 (develop), 18-24 (apply) |
| **Testing & Debugging** | Finding and fixing errors | Sessions 7-9 (introduce), 10-17 (develop), 18-24 (apply) |
| **Reusing & Remixing** | Building on existing work | Sessions 15-16 (functions), 19 (remix) |
| **Abstracting & Modularising** | Breaking into manageable parts | Sessions 15-17 (focus), 20-24 (apply) |

### Computational Thinking Perspectives

| Perspective | Indicator | Assessment Method |
|-------------|-----------|-------------------|
| **Expressing** | "I can create" | Project complexity and originality |
| **Connecting** | "I can collaborate" | Multiplayer projects, peer feedback |
| **Questioning** | "I can inquire" | Questions asked, problem-solving approaches |

# MODULE 1: FOUNDATIONS
## Sessions 1-4 | Focus: Interface, Events, Agent Basics

---

### Session 1: Welcome to Minecraft Education & Code Builder

**Duration:** 45 minutes

**Learning Objectives:**
- Navigate Minecraft Education Edition interface
- Access and understand Code Builder with MakeCode
- Execute first Agent command using blocks

**CT Concepts:** Introduction to Sequence
**CT Practices:** Experimenting and iterating
**CSTA Standards:** 1B-CS-01, 1B-AP-08

**Session Flow:**

| Time | Phase | Activity | Description |
|------|-------|----------|-------------|
| 0-5 min | Hook | "Coding in Minecraft?" | Discussion: How can we make Minecraft do things automatically? Introduce the Agent |
| 5-12 min | Setup | Account & World | Log in to Minecraft Education; create flat creative world; basic controls review |
| 12-20 min | Introduce | Code Builder Tour | Press 'C' to open Code Builder; explore MakeCode interface (blocks palette, workspace, Agent commands) |
| 20-35 min | Create | First Agent Programme | Guided: "On chat command 'go'" → Agent move forward. Test in-world. Add more movements |
| 35-42 min | Experiment | Block Exploration | Students try different Agent blocks (turn, move back, place block) |
| 42-45 min | Reflect | Exit Ticket | "What is the Agent? What is one command it can do?" |

**Key Vocabulary:**
- Agent (coding helper robot)
- Code Builder
- MakeCode
- Block (coding block)
- Chat command

**Key Concepts:**
- Agent as programmable entity
- MakeCode block-based interface
- Chat commands as event triggers
- Sequential execution

**Differentiation:**
- **Support:** Paired setup, visual guide handout
- **Extension:** Try multiple Agent commands in sequence

**Assessment:**
- Can student open Code Builder?
- Can student make Agent move with chat command?

### Session 2: Sequences - Building with Code

**Duration:** 45 minutes

**Learning Objectives:**
- Understand sequence as ordered steps
- Create multi-step Agent programmes
- Predict code execution before running

**CT Concepts:** Sequence
**CT Practices:** Experimenting and iterating
**CSTA Standards:** 1B-AP-08, 1B-AP-10

**Session Flow:**

| Time | Phase | Activity | Description |
|------|-------|----------|-------------|
| 0-5 min | Review | Recap Session 1 | Quick review: Agent, Code Builder, chat commands |
| 5-12 min | Unplugged | Sequence Challenge | Students write step-by-step instructions to draw a square on paper; discuss order importance |
| 12-18 min | Connect | Sequence in MakeCode | Demonstrate: Agent builds a line of 4 blocks. Order matters! |
| 18-32 min | Create | Build a Wall | Guided: Programme Agent to build a wall (move, place block, repeat sequence). Students predict before running |
| 32-40 min | Experiment | Modify & Extend | Change wall height/length; try different block types |
| 40-45 min | Reflect | Sequence Discussion | "Why does order matter? What happened when steps were wrong?" |

**Key Concepts:**
- Sequential execution (top to bottom)
- Agent place block command
- Combining movement and action
- Prediction before execution

**Common Misconceptions:**
- Agent executes all at once (no, sequential)
- Order doesn't matter (it does!)

**Assessment:**
- Wall builds correctly in sequence
- Can explain why order matters

**Take-home Challenge:**
- Think about what you'd like to build with code

# MODULE 2: SPATIAL THINKING
## Sessions 5-6 | Focus: Coordinates (X, Y, Z)

---

### Session 5: Introduction to 3D Coordinates

**Duration:** 45 minutes

**Learning Objectives:**
- Understand 3D coordinate system (X, Y, Z)
- Read and interpret position coordinates
- Use coordinates to position Agent and blocks

**CT Concepts:** Coordinates, Sequence
**CT Practices:** Abstracting and modularising
**CSTA Standards:** 1B-AP-08, 1B-DA-06
**Mathematics Integration:** Primary 4-5 coordinate geometry

**Session Flow:**

| Time | Phase | Activity | Description |
|------|-------|----------|-------------|
| 0-5 min | Hook | "Where Am I?" | Show coordinates display in Minecraft; discuss what numbers mean |
| 5-12 min | Unplugged | 3D Grid Activity | Use grid paper/blocks to understand X (left-right), Y (up-down), Z (forward-back) |
| 12-18 min | Introduce | Coordinates in Minecraft | X = East/West, Y = Up/Down, Z = North/South. Enable coordinate display; walk around and observe changes |
| 18-30 min | Create | Teleport Programme | Code: On chat command "tp" → Agent teleport to specific coordinates. Experiment with different positions |
| 30-40 min | Practice | Coordinate Challenges | Find the treasure: Given coordinates, teleport Agent there. Reverse: Find coordinates of specific locations |
| 40-45 min | Reflect | Real-World Coordinates | Discuss GPS, maps, 3D games - all use coordinate systems |

**Key Concepts:**
- X axis (East-West, left-right)
- Y axis (Up-Down, height)
- Z axis (North-South, forward-back)
- Absolute vs relative positions
- Teleport command

**Mathematics Connection:**
- 3D coordinate geometry
- Positive and negative numbers
- Spatial reasoning

**Assessment:**
- Can read current position coordinates
- Can teleport Agent to given coordinates
- Understands X, Y, Z axes

### Session 7: Introduction to Loops

**Duration:** 45 minutes

**Learning Objectives:**
- Understand why loops make code efficient
- Use "repeat" loops for counted iteration
- Compare code with and without loops

**CT Concepts:** Loops, Sequence
**CT Practices:** Abstracting and modularising
**CSTA Standards:** 1B-AP-10, 1B-AP-11

**Session Flow:**

| Time | Phase | Activity | Description |
|------|-------|----------|-------------|
| 0-5 min | Hook | Efficiency Problem | Show: Agent builds 10-block wall with 10 commands vs with loop. Which is better? |
| 5-12 min | Unplugged | Repeat Instructions | Write instructions to draw a square. Compare "move, turn, move, turn..." vs "repeat 4 times: move, turn" |
| 12-18 min | Introduce | Repeat Block | Demonstrate MakeCode "repeat" block. Code inside runs specified number of times |
| 18-30 min | Create | Loop Building | Agent builds a wall using loop: repeat 10 times (place block, move forward) |
| 30-38 min | Experiment | Different Patterns | Change loop count; nest loops for 2D patterns |
| 38-43 min | Compare | With vs Without | Count blocks used with loop vs without. Discuss efficiency |
| 43-45 min | Reflect | Loop Benefits | "Why are loops useful? When would you use them?" |

**Key Concepts:**
- Repeat (count-controlled) loop
- Loop body (code inside)
- Iteration count
- Code efficiency

**Key Vocabulary:**
- Loop, Repeat, Iteration
- Loop body

**Assessment:**
- Loop creates correct pattern
- Can explain why loops are efficient
- Can modify loop count appropriately
